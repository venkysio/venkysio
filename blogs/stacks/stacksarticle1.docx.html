STACKS

Stacks are common in everyday life, such as a stack of books, a stack of dishes, or a stack of chairs. A data structure stack is very similar to these examples (Fig. a).

                                        Stack of books        Stack of chairs        Stack of cups

                                                Fig. a Sample real world stacks

If you look at a stack of books on a table, you can easily add a new book at the top, and you can easily remove the topmost book compared to the books lying in the middle or at the bottom. A stack can also be accessed only at its topmost element, while other intermediate elements cannot be directly accessed. Elements can only be added or removed from one end, called its top.

In linear data structures, such as arrays and linked lists, we can insert or delete elements at any position in the list, i.e., at the beginning, at the end, or somewhere in between.

A stack is a restricted list wherein insertions and deletions are generally done only at the top. It has a data member, typically known as the 'top', which points to the foremost element in it. Pushing an item onto the stack or popping it off are the two basic operations that can be performed. Unlike other data structures, getting access to elements from intermediate positions (other than the top) is not possible in stacks.

Given a stack S = (a1, a2, ..., an). We say that as a1 is the bottommost element, an is on top of the stack, and the element ai+1 is said to be on the top of ai, 1 < i ≤ n.

In Fig. b, S = (A, B, C), where A is the bottommost element and C is the topmost element.

  1. Primitive Operations

Push, pop, and getTop are the three basic stack operations. Other operations that can be implemented on a stack include stack_initialization, stack_empty, and stack_full. The stack_initialization operation prepares the stack for use and sets it to a vacant state. The stack_empty operation checks if the stack is empty - this is important to prevent attempting to pop an element from an empty stack which would be an error condition (stack underflow). Stacks should have infinite capacity so that the subsequent elements can always be pushed - however memory capacity is finite, so we must check for a full stack before doing a push (stack overflow).

The GetTop stack operation returns the top element of the stack without popping it. Stack operations include traversing the stack, counting the number of elements in the stack, and copying the stack. Let us quickly review these operations.

  1. The Push method inserts an element at the top of the stack
  2. A pop operation removes an element from the top of the stack
  3. GetTop reads (only reads, does not delete) an element from the top of the stack
  4. Stack_initialization - creates an empty stack
  5. Empty - determines if the stack is empty
  6. Full - checks if the stack is full

Push

Pushing puts an element on top of the stack. The recently added element is always on top. Before pushing, we have to make sure there is room for a new element (Fig. c).

Fig. c The push operation

When there is no room on the stack to accommodate the new element, the stack is said to be full (Fig. d). When the stack is full, it is said to be in overflow state, which means that no element can be added. Fig. c shows how pushing changes the top because the newly inserted element becomes the topmost element.

Before every pop, we must make sure the stack is not empty. After deleting the last element from the stack, the stack should be set to empty.

GetTop

The key difference between pop and getTop is that getTop does not modify the variable top. If the stack is empty, it signals an underflow error.
